# VoiceNote → Obsidian Pipeline

Capture voice notes and text messages from **WhatsApp** or **Telegram**, transcribe them locally with [faster-whisper](https://github.com/SYSTRAN/faster-whisper), reformulate them with **Groq AI** (Llama 3.1 70B), and archive the result as a Markdown note in your **Obsidian vault** — automatically categorised.

```
WhatsApp / Telegram
       ↓
 Channel Adapter
       ↓
 VoiceQueue (FIFO)     ← voice only
       ↓
  faster-whisper        ← local transcription, CPU
       ↓
   Groq Llama 3.1       ← reformulation + categorisation
       ↓
 Obsidian Vault (.md)   ← YAML frontmatter + body
```

## Requirements

| Dependency | Version |
|---|---|
| Node.js | ≥ 20 |
| Python | ≥ 3.10 |
| ffmpeg | any recent |
| Groq API key | [console.groq.com](https://console.groq.com) (free tier) |

**WhatsApp**: requires a dedicated secondary number (personal account risk — eSIM or spare phone recommended).  
**Telegram**: create a bot via [@BotFather](https://t.me/BotFather), get your user ID via [@userinfobot](https://t.me/userinfobot).

## Installation

```bash
git clone https://github.com/TheArabicMonster/voicenote-obsidian-pipeline.git
cd voicenote-obsidian-pipeline

# Node dependencies
npm install

# Python dependencies (faster-whisper)
pip install -r requirements.txt

# Interactive configuration wizard
npm run setup
# or: node bin/setup.js
```

The wizard guides you through 5 steps:

1. Channel selection (WhatsApp and/or Telegram)
2. Channel credentials (number or bot token + user ID)
3. Obsidian vault path + initial categories
4. Whisper model and language + Groq API key
5. Summary and confirmation

It writes `config.yaml`, `.env`, creates the vault folder structure, and initialises `categories.json`.

## How it works

### Voice message flow

1. User sends a voice note → bot replies with an immediate acknowledgment
2. Job queued in the **FIFO VoiceQueue** (one Whisper process at a time, CPU-safe)
3. Audio saved as `.ogg` → converted to 16 kHz mono WAV via `ffmpeg`
4. `transcribe.py` runs `faster-whisper` locally → returns `{"text":"...", "language":"fr", "duration":12.3}`
5. Transcription passed to **Groq API** (Llama 3.1 70B) with the active category list injected into the system prompt
6. AI returns structured JSON: title, reformulated content, category, tags, priority, reasoning
7. Note written to vault as `YYYY-MM-DD_HH-MM_slug.md` with YAML frontmatter
8. Temporary audio files deleted

### Text message flow

Same as steps 5–8 above, bypassing the queue (no transcription needed).

### New category proposal

When the AI determines that no existing category fits, it sets `"nouvelle_categorie": true` in its response. The bot then:

1. Sends a confirmation prompt to the user (OUI / NON)
2. Saves the proposal to `pending.json` (crash-safe persistence)
3. If **OUI**: creates the category folder, writes the note, updates `categories.json`
4. If **NON**: writes the note to `_Inbox` with `categorie_suggeree` in the frontmatter
5. Proposals expire after 10 minutes (configurable) → automatically filed to `_Inbox`
6. Multiple proposals drain in FIFO order: each OUI/NON surfaces the next pending item

## Configuration reference

### `config.yaml`

```yaml
channels:
  whatsapp:
    enabled: true
    allowedNumber: "+41791234567"   # Only this number can send messages
    sessionPath: ".wwebjs_auth"     # WhatsApp session persistence folder
  telegram:
    enabled: false
    allowedUserId: "123456789"      # Only this Telegram user ID is accepted

vault:
  path: "/home/ubuntu/obsidian-vault"
  categoriesFile: "categories.json"
  inboxFolder: "_Inbox"
  inboxTag: "a-classer"

whisper:
  model: medium                     # tiny | base | small | medium | large-v3
  language: fr                      # ISO 639-1 language code

pipeline:
  categoryApprovalTimeoutMin: 10    # Minutes before a pending proposal expires
  audioTmpPath: "./tmp/audio"
```

### `.env`

```env
WA_ENABLED=true
WA_ALLOWED_NUMBER=+41791234567
TG_ENABLED=false
TG_BOT_TOKEN=<botfather-token>
TG_ALLOWED_USER_ID=123456789
VAULT_PATH=/home/ubuntu/obsidian-vault
CATEGORIES_FILE=/home/ubuntu/obsidian-vault/categories.json
PENDING_FILE=/home/ubuntu/voicenote-obsidian-pipeline/pending.json
GROQ_API_KEY=gsk_...
WHISPER_MODEL=medium
WHISPER_LANGUAGE=fr
CATEGORY_APPROVAL_TIMEOUT_MIN=10
```

## Vault note format

Each archived note follows this structure:

```markdown
---
date: 2025-09-15T14:32:00
source: telegram-vocal
categorie: Projets/Tech
tags: [web, backend]
priorite: haute
statut: nouveau
raisonnement_ia: "L'idée porte sur une architecture API, proche des projets tech existants."
---

# Refactoriser l'authentification JWT

Migrer vers des tokens à courte durée de vie avec rotation automatique.

---
*Transcription originale :* "euh je voulais noter qu'il faudrait revoir le système de tokens JWT..."
```

Notes filed to `_Inbox` receive `statut: a-classer`, the `a-classer` tag, and `categorie_suggeree` if the AI proposed a new category.

## Whisper models

| Model | Size | Speed | Quality |
|---|---|---|---|
| tiny | 75 MB | fastest | basic |
| base | 145 MB | fast | good |
| small | 465 MB | moderate | good |
| **medium** | 1.5 GB | moderate | **recommended** |
| large-v3 | 3 GB | slow | best |

The model is downloaded on first use and cached by faster-whisper.
